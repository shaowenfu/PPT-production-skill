# PPT Assistant Skill & Workflow Spec

## 核心目标
本 Skill 旨在通过“Agent-Centric”模式，驱动 Python 脚本完成从原始大纲到成品级 PPT 的端到端生产。

## 7 步工作流协议
1.  **项目初始化**：调用 `project_init.py` 建立 `state.json` 环境。
2.  **大纲提取**：Agent 提取深度大纲并保存为 `outline.md`。
3.  **分页规划**：Agent 规划 25 页逻辑（默认规模），分配 A/B 类。
    - **A/B 比例**：B 类占 20%-40%。总页数 > 4 至少 2 个 B 类。
4.  **深度文案稿**：调用 `slide_draft_generate.py` 批量生成。每批 5 页。
5.  **视觉导演**：Agent 编写全页面 `prompts.json`。
    - **导演契约**：提示词必须包含具体中文文案、构图指令和光影风格。
6.  **资产生成**：调用 `visual_asset_generate.py` 使用阿里云模型出图。
7.  **最终组装**：调用 `ppt_assemble.py` 完成“图像即页面”封装。

## 状态机不变量
- **A 类（逻辑型）**：强调结构化知识、留白、文字可读性。
- **B 类（情感型）**：强调冲击力、意象、金句和视觉氛围。
- **全页面出图**：所有页面均通过图片生成，PPTX 仅作为容器。

## 失败与重试作用域
- 文案不准：回退到第 4 步（Draft）。
- 风格不对：回退到第 5 步（Prompt）。
- 比例不合：回退到第 3 步（Plan）。
